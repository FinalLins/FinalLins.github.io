1. 定义

  希尔排序(Shell Sort)是插入排序的一种又缩小增量排序（Diminishing Increment Sort），是直接插入排序算法的一种更高效的改进版本。希尔排序是非稳定排序算法。该方法因D.L.Shell于1959年提出而得名。希尔排序是把记录按下标的一定增量分组，对每组使用直接插入排序算法排序；随着增量逐渐减少，每组包含的关键词越来越多，当增量减至1时，整个文件恰被分成一组，算法便终止。

2. 原理

  先将整个待排序的记录序列分割成为若干子序列分别进行直接插入排序，待整个序列中的记录“基本有序”时，再对全体记录进行依次直接插入排序。

  • 选择一个增量序列 t1，t2，……，tk，其中 ti > tj, tk = 1，($t_n = (t_{n-1}) / 2$）；
  • 按增量序列个数 k，对序列进行 k 趟排序；
  • 每趟排序，根据对应的增量 ti，将待排序列分割成若干长度为 m 的子序列，分别对各子表进行直接插入排序。仅增量因子为 1 时，整个序列作为一个表来处理，表长度即为整个序列的长度。

希尔排序也是插入排序的一种，它外层多了增量gap去控制外层循环。内层还是插入排序的核心思想。

3. 算法分析

  希尔排序的复杂度和增量序列是相关的，不同gap序列的设计对应不同的复杂度。

  • {$1,2,4,8,…$}这种序列并不是很好的增量序列，使用这个增量序列的时间复杂度（最坏情形）是$O(n^2)$
  • Hibbard提出了另一个增量序列{$1,3,7，…,2^{k-1}$}，这种序列的时间复杂度(最坏情形)为$O(n^{1.5})$
  • Sedgewick提出了几种增量序列，其最坏情形运行时间为$O(n^{1.3})$,其中最好的一个序列是{$1,5,19,41,109…$}

4. 稳定性

  虽然插入排序是稳定排序，但是希尔排序在插入的时候是跳跃性插入的，有可能破坏稳定性。所以希尔排序是一个不稳定的排序。

5. 代码实现

  学过直接插入排序的同学，写希尔排序应该很简单了。

class Shell {
    public static final String name = "希尔排序";

    public static int[] sort10(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }

        printHeader(name, "1.0", arr);
        int length = arr.length;
        int temp, gap, count = 0;
        for (gap = length >> 1; gap > 0; gap >>= 1) {
            for (int i = gap; i < length; i++) {
                temp = arr[i];
                int j = i - gap;
                while (j >= 0 && arr[j] > temp) {
                    arr[j + gap] = arr[j];
                    j -= gap;
                    count++;
                }
                arr[j + gap] = temp;
                printLoop(i - gap, arr, "增量gap :" + gap);
            }
        }
        printFooter(count);
        return arr;
    }

    public static int[] sort11(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }

        printHeader(name, "1.1", arr);
        int length = arr.length;
        int temp, gap = 1, count = 0;
        while (gap < length / 3) {
            gap = gap * 3 + 1;
        }
        for (; gap > 0; gap /= 3) {
            for (int i = gap; i < length; i++) {
                temp = arr[i];
                int j = i - gap;
                while (j >= 0 && arr[j] > temp) {
                    arr[j + gap] = arr[j];
                    j -= gap;
                    count++;
                }
                arr[j + gap] = temp;
                printLoop(i - gap, arr, "增量gap :" + gap);
            }
        }
        printFooter(count);
        return arr;
    }
}

7. 总结

  从所有数据观察，区别不大。相对于插入排序而言，比较较远距离的数据，使得数据移动跨过多个元素，进行一次比较可能会消除多个元素的交换。

对于中等大小的数据，希尔排序的运行时间是可以接受的。它的代码量不大，也不需要使用额外的内存空间。要注意的是它的不稳定性，如果对稳定性有要求那就不能选择希尔排序了。
