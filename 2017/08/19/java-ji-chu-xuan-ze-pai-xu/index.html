1. 定义

  选择排序(Selection Sort)是一种简单直观的排序算法。它的工作原理：首先在未排序序列中找到最小（大）元素，存放到排序序列的起始位置，然后，再从剩余未排序元素中继续寻找最小（大）元素，然后放到已排序序列的末尾。以此类推，直到所有元素均排序完毕。简单选择排序属于选择排序的一种。

平均时间复杂度 最好的时间复杂度 最坏的时间复杂度 空间复杂度 排序方法 稳定性
$O_{(n^2)}$ $O_{(n^2)}$ $O_{(n^2)}$ $O_{(1)}$ 内部排序(In-place) 不稳定

2. 算法分析

  选择排序是表现最稳定的排序算法之一（这是的稳定不是指算法的稳定性）。因为无论什么数据进入时间复杂度都是$O_{(n^2)}$。所以用到它的时候，数据规模越小越好。所以最好的情况和最坏的情况时间复杂度都是一样的。时间复杂度为 $T_{(n)} = O_{(n^2)}$，空间复杂度 $S_{(n)} = O_{(1)}$。（去掉低阶项，去掉常数项，去掉高阶项的常参得到）

3. 稳定性

  选择排序是给每个位置选择当前元素最小的，比如给第一个位置选择最小的，在剩余元素里面给第二个元素选择第二小的，依次类推，直到第n-1个元素，第n个元素不用选择了，因为只剩下它一个最大的元素了。那么，在一趟选择，如果一个元素比当前元素小，而该小的元素又出现在一个和当前元素相等的元素后面，那么交换后稳定性就被破坏了。比较拗口，举个例子，序列5 8 5 2 9，我们知道第一遍选择第1个元素5会和2交换，那么原序列中两个5的相对前后顺序就被破坏了，所以选择排序是一个不稳定的排序算法。

4. 代码实现

  在大学里面刚接触C语言的时候在老师好那里学会了一种排序算法(sort10(int[]))，当时对冒泡和选择搞得不是很清楚，它俩很像。

下面有部分方法在冒泡排序文章最后面可以找到。

class Select {
    public static final String name = "选择排序";

    public static void main(String[] args) {
        Select.sort10(resetArr());
        Select.sort11(resetArr());
        Select.sort12(resetArr());
    }

    public static int[] sort10(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }

        printHeader(name, "1.0", arr);
        int length = arr.length;
        int count = 0;
        for (int i = 0; i < length - 1; i++) {
            for (int j = i + 1; j < length; j++) {
                if (arr[i] > arr[j]) {
                    swap(i, j, arr);
                }
                count++;
            }
            printLoop(i, arr);
        }
        printFooter(count);
        return arr;
    }

    public static int[] sort11(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }

        printHeader(name, "1.1", arr);
        int count = 0;
        int length = arr.length;
        int min;
        for (int i = 0; i < length - 1; i++) {
            min = i;
            for (int j = i + 1; j < length; j++) {
                if (arr[min] > arr[j]) {
                    min = j;
                }
                count++;
            }
            swap(i, min, arr);
            printLoop(i, arr);
        }

        printFooter(count);
        return arr;
    }

    public static int[] sort12(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }

        printHeader(name, "1.2", arr);
        int count = 0;
        int length = arr.length;
        int min;
        for (int i = 0; i < length - 1; i++) {
            min = i;
            for (int j = i + 1; j < length; j++) {
                if (arr[min] > arr[j]) {
                    min = j;
                }
                count++;
            }
            if (min == i) {
                //数据不需要交换
                printLoop(i, arr, "i 就是最小值，不需要交换");
                continue;
            }
            swap(i, min, arr);
            printLoop(i, arr);
        }

        printFooter(count);
        return arr;
    }

}
  • sort10 方法用了选择排序的思想又跟冒泡排序很像。
  • sort11 是最常规的选择排序的写法。
  • sort12 是对sort11进行了优化，对内存循环获取的最小值和原值进行判断，如果相等说明i就是最小值，不需要交换。

6. 总结

  从三组测试结果来看，第三种方法是最高效的选择排序，当然上述测试数据只是在我的环境下的单一测试的结果。并不能代表宏观的测试结果。选择排序跟冒泡其实很像，相对于冒泡排序减少了交换的次数。

当数据量不大，且对稳定性没有要求的时候，适用于选择排序。
