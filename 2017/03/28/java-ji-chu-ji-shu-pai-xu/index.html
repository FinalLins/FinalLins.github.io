1. 概述

  计数排序是一个非基于比较的排序算法，该算法于1954年由 Harold H. Seward 提出。它的优势在于在对一定范围内的整数排序时，它的复杂度为Ο(n+k)（其中k是整数的范围），快于任何比较排序算法。当然这是一种牺牲空间换取时间的做法，而且当O(k)>O(nlog(n))的时候其效率反而不如基于比较的排序（基于比较的排序的时间复杂度在理论上的下限是O(nlog(n)), 如归并排序，堆排序），计数排序是稳定排序。

思路如下：

  1. 原数组A，遍历获取最小值min和最大值max；
  2. 创建临时数组B，长度为max - min + 1；
  3. 遍历待排序数组，它的值value - min作为临时数组B的下标，并且值+1（B[value - min]++）；
  4. 最后遍历临时数组B将它展开赋值给原数组。

2. 代码实现

2.1 写法一

Class Count {

    public static final String name = "计数排序";

    public static int[] sort10(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }
        printHeader(name, "1.0", arr);

        //1.遍历获取最小值``min``和最大值``max``；
        int length = arr.length;
        int min = arr[0], max = arr[0];
        for (int i = 0, value; i < length; i++) {
            value = arr[i];
            if (value < 0) {
                throw new RuntimeException("计数排序只支持大于等于0的整数");
            }
            if (value > max) {
                max = value;
            }

            if (value < min) {
                min = value;
            }
        }

        System.out.println(min + "--" + max);
        //2.创建临时数组B，长度为``max - min + 1``；
        int bLen = max - min + 1;
        int[] B = new int[bLen];

        //3.遍历待排序数组，它的值``value - min``作为临时数组B的下标，并且值+1（``B[value - min]++``）
        for (int i = 0, value, index; i < length; i++) {
            value = arr[i];
            index = value - min;
            B[index]++;
        }
        printArray(B);
        //4.最后遍历临时数组B将它展开赋值给原数组。
        for (int i = 0, value, index = 0; i < bLen; i++) {
            value = B[i];
            while (value-- != 0) {
                arr[index++] = i + min;
            }
        }
    }
}

我在学习计数排序的时候，对第4个步骤也理解了很久。（可能属于比较笨的）
我们拿一组数据来测试下，对着数据就很好理解了。

    public static void main(String[] args) {
        int[] arr = new int[]{6, 2, 10, 5, 7, 8, 2};
        Count.sort10(arr);
    }
    //输出如下
    ---------------------------------------------
    计数排序v1.0 : [6, 2, 10, 5, 7, 8, 2]
    2--10
    [2, 0, 0, 1, 1, 1, 1, 0, 1]
    [2, 2, 5, 6, 7, 8, 10]
    排序结束
    ---------------------------------------------

[2, 0, 0, 1, 1, 1, 1, 0, 1] //
[0, 1, 2, 3, 4, 5, 6, 7, 8] //编号（其实就是B数组的下标）
我们看第三步 B数组下标怎么计算出来的，是通过原数组的值减去最小值。(value - min)
那么反过来原数组的值是不是就等于 下标加上最小值呢？ value = i + min

    public static int[] sort11(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }

        int length = arr.length;
        printHeader(name, "1.1", arr);

        //1.遍历获取最小值``min``和最大值``max``；
        int min = arr[0], max = arr[0];
        for (int i = 0, value; i < length; i++) {
            value = arr[i];
            if (value < 0) {
                throw new RuntimeException("计数排序只支持大于等于0的整数");
            }

            if (value > max) {
                max = value;
            }

            if (value < min) {
                min = value;
            }
        }
        System.out.println(min + "--" + max);
        //2.创建临时数组B，长度为``max - min + 1``；
        int bLen = max - min + 1;
        int[] B = new int[bLen];

        //3.遍历待排序数组，它的值``value - min``作为临时数组B的下标，并且值+1（``B[value - min]++``）
        for (int i = 0, value, index; i < length; i++) {
            value = arr[i];
            index = value - min;
            B[index]++;
        }
        printArray(B);

        //4.对所有的计数累加
        for (int i = 1; i < cLen; i++) {
            B[i] += B[i - 1];
        }
        printArray(B);

        //5.逆序遍历源数组，找到元素正确位置，逆序是为了保证不破坏稳定性
        int[] newArr = new int[length];
        for (int i = length - 1, index, value; i >= 0; i--) {
            value = arr[i];
            index = B[value - min] - 1;
            newArr[index] = value;
            B[value - min]--;
        }
        printArray(newArr);
        return newArr;
    }
