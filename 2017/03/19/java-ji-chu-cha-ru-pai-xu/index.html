1. 定义

  直接插入排序（Insertion sort）是一种简单直观且稳定的排序算法。基本思想：把待排序的记录按其关键码值的大小逐个插入到一个已经排好序的有序序列中，直到所有的记录插入完为止，得到一个新的有序序列。直接插入排序是插入排序的一种。

2. 原理

  把 n 个待排序的元素看成为一个有序表和一个无序表。开始时有序表中只包含 1 个元素，无序表中包含有 n - 1 个元素，排序过程中每次从无序表中取出第一个元素，将它插入到有序表中的适当位置，使之成为新的有序表，重复 n - 1 次可完成排序过程。

3. 算法分析

  插入排序的平均时间复杂度是$O(n^2)$,最好情况是$O(n)$,最差情况是$O(n^2)$,是一种稳定的排序方法，空间复杂度是$O(1)$。大部分已排序好时较好使用。

4.1 教科书写法

  1. 从前往后依次查找插入位置；
  2. 将插入位置之后i之前元素全部往后移一格；
  3. 插入指定元素。
    /**
     * 教科书写法
     *
     * @param arr
     * @return
     */
    public static int[] sort10(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }
        printHeader(name, "1.0", arr);
        int length = arr.length;
        int count = 0;
        int i, j, k;
        for (i = 1; i < length; i++) {          //从下标1开始
            //在有序区间中为a[i]找插入位置（倒序查找），这种方法找出来的位置是 [j + 1]
            for (j = i - 1; j >= 0; j--) {
                count++;
                if (arr[j] < arr[i]) {
                    break;
                }
            }

            //如果j +1 == i，说明arr[i] 大于有序列表里面所有数据，应该插入末尾，其实就是arr[i]本身自己的位置。
            if (j != i - 1) {
                //给a[i]腾地方
                int temp = arr[i];
                for (k = i - 1; k > j; k--) {
                    arr[k + 1] = arr[k];
                    count++;
                }
                arr[j + 1] = temp;
            }
            printLoop(i - 1, arr);
        }
        printFooter(count);
        return arr;
    }

    public static int[] sort11(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }
        printHeader(name, "1.1", arr);
        int length = arr.length;
        int count = 0;
        int i, j, k;
        for (i = 1; i < length; i++) {        //从下标1开始
            //在有序区间中为arr[i]找插入位置（顺序查找），位置[j]
            for (j = 0; j < i; j++) {
                count++;
                if (arr[j] > arr[i]) {
                    break;
                }
            }

            //如果j == i，说明arr[i] 大于有序列表里面所有数据，应该插入末尾，其实就是arr[i]本身自己的位置。
            if (j != i) {
                int temp = arr[i];           //保存待插入元素
                // arr[j + 1] ~ arr[i - 1] 数据整体往右移一个下标
                for (k = i ; k > j; k--) {
                    arr[k] = arr[k - 1];
                    count++;
                }
                arr[j] = temp;              //把arr[i] 插入到计算好的位置 j
            }
            printLoop(i - 1, arr);
        }
        printFooter(count);
        return arr;
    }

同学们能看出两种方法有说明区别么？

就是在查找待插入元素的位置的时候的一个是顺序一个是倒序的区别。
计算出来的 j 的值不一样。

4.2 优化版本2.0

  上面的这种方式第一次查找插入位置的时候需要将这些元素遍历一遍，移动的时候又得遍历一遍，进行了重复的运算，如果第一步寻找插入位置时，我们从后往前寻找插入位置的话，第一步和第二步就可以一起进行：

    public static int[] sort20(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }
        printHeader(name, "2.0", arr);
        int length = arr.length;
        int count = 0;
        int i, j;
        for (i = 1; i < length; i++) {
            int temp = arr[i];              //无序列表第一个元素（待插入到有序列表中的元素）
            j = i - 1;                      //有序列表最后一个元素
            for (; j >= 0; j--) {
                count++;
                if (arr[j] > temp) {
                    arr[j + 1] = arr[j];    //数据往后移动
                } else {
                    break;
                }
            }
            arr[j + 1] = temp;              //插入到有序列表中，有序列表元素+1
            printLoop(i - 1, arr);
        }
        printFooter(count);
        return arr;
    }

    public static int[] sort21(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }
        printHeader(name, "2.1", arr);
        int length = arr.length;
        int count = 0;
        int i, j;
        for (i = 1; i < length; i++) {
            int temp = arr[i];              //无序列表第一个元素（待插入到有序列表中的元素）
            j = i - 1;                      //有序列表最后一个元素
            for (; j >= 0 && arr[j] > temp; j--) {
                arr[j + 1] = arr[j];        //数据往后移动
                count++;
            }
            arr[j + 1] = temp;              //插入到有序列表中，有序列表元素+1
            printLoop(i - 1, arr);
        }
        printFooter(count);
        return arr;
    }

4.3 优化版本3.0（二分法查找位置）

  可以看到，在寻找插入位置的时候，是一个一个进行比较的，这样的效率着实很低。所以可以将查找位置部分进行一下优化，不妨利用二分查找来找到待插入位置，然后再进行插入就好了。

    /**
     * 通过二分法思路查找元素item插入到有序列表arr的位置
     * 比如 7 插入到 [ 1, 3, 5, 9, 10] return 3
     *
     * @param arr
     * @param item
     * @param low
     * @param high
     * @return
     */
    public static int insertBinarySearch(int arr[], int item, int low, int high) {
        if (high <= low) {
            return item >= arr[low] ? low + 1 : low;
        }

        int mid = low + ((high - low) >> 1);

        if (item == arr[mid]) {
            return mid + 1;             //值相同，往后插入
        }
        if (item > arr[mid]) {
            return insertBinarySearch(arr, item, mid + 1, high);
        } else {
            return insertBinarySearch(arr, item, low, mid - 1);
        }
    }

    public static int[] sort30(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }
        printHeader(name, "3.0", arr);
        int length = arr.length;
        int count = 0;
        int i, j, k;
        for (i = 1; i < length; i++) {
            int temp = arr[i];          //无序列表第一个元素（待插入到有序列表中的元素）
            j = i - 1;                  //有序列表最后一个元素
            k = insertBinarySearch(arr, temp, 0, j);
            while (j >= k) {
                arr[j + 1] = arr[j];    //数据往后移动
                j--;
                count++;
            }
            arr[j + 1] = temp;          //插入到有序列表中，有序列表元素+1
            printLoop(i - 1, arr);
        }
        printFooter(count);
        return arr;
    }

4.4 哨兵版本

  网上有很多版本都提到了哨兵，但是很多人只是把代码贴上去，并没有讲到哨兵它不进行排序。哨兵的两个作用：

  1. 临时缓存待插入的元素；
  2. 防止数组下标越界。（相对于普通写法就可以去掉 j >= 0的判断，优化效率）

这种排序方式有个严重的弊端，arr[0]中不存储有用的元素，什么意思呢？

[3, 2, 6, 7, 1, 9] (sort)==> [9, 1, 2, 6, 7, 9]

首位（哨兵位），它不会进行排序。

    public static int[] sort40(int[] arr) {
        if (arr == null || arr.length == 0) {
            return arr;
        }
        int length = arr.length;
        int i, j;
        for (i = 2; i < length; i++) {
            arr[0] = arr[i];            //哨兵的作用之一，缓存待插入的元素
            for (j = i - 1; arr[j] > arr[0]; j--) {
                arr[j + 1] = arr[j];
            }
            arr[j + 1] = arr[0];
        }
        return arr;
    }

5. 总结

直接插入排序对小数据量或者基本有序时十分高效。

适用于数据量不大，对算法的稳定性有要求，且数据局部或者整体有序的情况。
